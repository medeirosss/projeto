UEM · Active Directory
Conectividade com o domínio
A senha é armazenada de forma criptografada no backend. Para produção, recomenda-se LDAPS; para homologação/clientes sem certificado, use LDAP 389.

UEM · IP Scope
Redes internas permitidas
O escopo é usado para decidir se a ação é local pelo backend ou externa via Endpoint Central / n8n.
Webhook · Alertas inbound
Recebimento seguro de alertas
O endpoint /api/actions/inbound-alert pode receber POST de ferramentas como ADAudit. Origens confiáveis podem enviar sem token; fora dessas redes, o header X-Centric-Webhook-Token é exigido.
Se proxy estiver habilitado, headers como X-Forwarded-For só serão aceitos quando a conexão vier de um proxy confiável.
Usuários · Acesso ao portal
Usuários e grupos autorizados do AD
A aplicação valida a senha diretamente no Active Directory e não armazena credenciais do usuário. Aqui ficam apenas usuários/grupos autorizados e suas roles.
Roles: admin = tudo; operator = alertas, ações e relatórios; viewer = dashboard e relatórios.
